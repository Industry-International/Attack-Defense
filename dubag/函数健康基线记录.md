## 后续 dubag 使用说明

1. 每次 dubag 前，运行 `git diff --name-only <基线HASH> HEAD` 获取变更文件列表
2. 仅对变更文件 + 变更文件引用的文件进行 dubag
3. 引用关系：读取变更文件中的 `function game:xxx` 和 `execute ... run function game:xxx` 调用
4. 如果某函数被标记为"无已知问题"但被变更文件依赖，dubag 时也应检查该函数是否因依赖变化而出现新问题
5. 在旧 dubag.md 中已修复的问题，在新报告中仅作简单提示
